# SoundLift Audio Booster Privacy Policy

Last updated: June 27, 2026

SoundLift Audio Booster is a Chrome extension that boosts audio volume for the active browser tab selected by the user. This privacy policy explains what data the extension does and does not collect.

## Summary

SoundLift Audio Booster does **not** collect, store, sell, share, or transmit personal data.

The extension works locally inside your browser. Audio processing, volume boost, Safe Mode, voice presets, bass boost, keyboard shortcuts, language settings, theme settings, and per-site profiles are handled locally.

## Data Collection

SoundLift Audio Booster does **not** collect any of the following:

- Personally identifiable information
- Health information
- Financial or payment information
- Authentication information
- Personal communications
- Location data
- Web browsing history
- User activity
- Website content
- Audio recordings
- Page text, images, videos, or links
- Analytics data
- Advertising identifiers
- Fingerprints or tracking identifiers

## How the Extension Works

SoundLift may request access to browser tabs and websites so it can detect and boost audio on the tab selected by the user.

The extension may use:

- `activeTab` to identify the tab selected by the user.
- `tabs` to read basic tab metadata such as tab ID, title, domain, audible state, and active state.
- `tabCapture` to capture and process audio from the selected active tab when direct media control is unavailable.
- `offscreen` to keep the local Web Audio processing engine running while the popup is closed.
- `storage` to save user settings locally in Chrome storage.
- Host permissions to detect HTML5 audio or video on websites where the user requests audio boosting.

These permissions are used only to provide the extension’s core function: boosting audio volume for the selected active tab.

## Local Storage

SoundLift may save the following settings locally in Chrome storage:

- Default boost level
- Maximum boost cap
- Safe Mode preference
- Voice Boost preference
- Bass Boost preference
- Theme preference
- Language preference
- Per-site volume profiles
- Diagnostics preference

These settings remain on the user’s device. They are not sent to any server.

## Audio Processing

SoundLift processes audio locally in the browser using Web Audio APIs.

SoundLift does **not**:

- Record audio
- Store audio
- Upload audio
- Analyze audio remotely
- Send audio to third parties

## Website Content

SoundLift does **not** read, store, sell, transmit, or modify website content for tracking or advertising purposes.

The extension may detect HTML5 `<audio>` and `<video>` elements only to apply audio boosting after the user requests it.

## No Tracking or Advertising

SoundLift does **not** use:

- Analytics
- Tracking scripts
- Fingerprinting
- Advertising SDKs
- Remote code
- Sponsored content injection
- Affiliate link injection

SoundLift does not insert ads into web pages.

## Remote Code

SoundLift does **not** load or execute remote code. All JavaScript, HTML, CSS, images, and other assets are packaged inside the extension.

## Third-Party Sharing

SoundLift does **not** sell, share, transfer, or disclose user data to third parties.

## Optional Donation Link

SoundLift may include an optional donation link inside the extension interface. The link opens only if the user clicks it. Visiting the donation page is optional and is handled by the external donation provider under its own privacy policy.

## Children’s Privacy

SoundLift does not knowingly collect data from children or any other users.

If this privacy policy changes, the updated version will be published in this repository.
